### 1. Цель работы

Целью данной кейс задачи является создание игры (программы), в процессе которой пользователь пытается угадать случайно выбранное компьютером число в диапазоне от 1 до 100.

Игра запрашивает у пользователя ввод целого числа, анализирует введённое значение на предмет соответствия условию "целое число", в случпе успешной проверки сравнивает введённое значение с числом, случайно выбранным компьютером, выводит подсказки для пользователя, затем выводит результат игры.

### 2. Основные функции:

Разработанная программа выполняет следующие функции:
* генерация случайно заданного числа в диапазоне от 1 до 100 с пощощью функции *randint()* модуля *random*;
* вывод приветственного сообщения, включающего объяснения правил игры и число попыток угадывания;
* запрос ввода числа;
* валидация введёного значения на соответствие типу "целое число";
* валидация введёного значения на соответствие указанному диапазону;

### 3. Интерфейс программы:

Интерфейс программы представляет собой взаимодействие с пользователем с помощью консольной строки.

### 4. Алгоритм работы программы:

1. Программа выводит приветственное сообщение.
2. Запускается цикл с предусловием.
3. Программа выводит запрос введения числа.
4. С помощью встроенной функции int() программа преобразовывает введённое значение в целое число.
5. В случае исключения ValueError, связанного с некорректностью ввода, программа выводит сообщение об ошибке и выводит повторный запрос ввода числа.
6. В случае успешного прохождения п.4 программа проверяет является ли число неотрицательным.
7. В случае отрицательного числа программа сообщает, что факториал для отрицательного числа определить невозможно и выводит повторный запрос ввода числа.
8. В случае успешного прохождения проверок программа вычисляет факториал введёного числа с помощью функции factorial() библиотеки math.
9. Программа выводит результат на экран.
10. Цикл с предусловием завершается.

### 5. Валидация и обработка исключений

В программе реализованы два вида проверки:

* 1.Проверка формата ввода
    * Программа проверяет возможно ли преобразование введёного значения в целое число с помощью конструкции try/except. Это позволяет обработать случаи введения пустого или некорректного значения, включая дробные числа, текст и спецсимволы.

* Проверка допустимости значения
    * После успешного преобразования введёного значения программа проверяет является ли число допустимым для дальнейшего использования. Допустимым считается только неотрицательное целое число, т.к. факториал отрицательных или нецелых чисел не определяется.

### 6. Пользовательский опыт

Программа реализована удобно для консольного решения, т.к. пользователю понятно, что необходимо ввести, ясно сформированы сообщения об ошибках, программа не завершается при неверном вводе, успешный результат отображается отдельно, повторный ввод осуществляется автоматически.

### 7. Производительность

Производительность обеспечивается использованием функции *factorial()* библиотеки *math*, оптимизированной для вычисления математических операций.

### 8. Расширяемость

В качестве будущих расширений программу можно дополнить как новыми возможностями, так и дополнительными проверками, например:

* ограничение минимального и максимального значения;
* запрет на введение слишком больших чисел;
* повторный расчёт без перезапуска программы;
* логирование ошибок ввода;
* вынесение проверок в отдельный модуль.
